# 株式会社須田製版 採用サイト TOP — コーディングデータ

デザインカンプ(SVG / 1600×4305px)を元にした1ページコーディングです。

## ディレクトリ構成

```
sudaseihanRecruit/
├─ index.html
├─ style.css            … サイト固有スタイル(このサイトのレイアウト本体)
├─ css/
│   ├─ layout.css       … リセット・共通パーツ(汎用・他案件共通/そのまま使用)
│   └─ common.css       … 汎用ユーティリティクラス(flex/margin等/そのまま使用)
├─ js/
│   └─ script.js        … 縮小フィット・ドロワー・スムーススクロール・モーダル
└─ images/              … カンプから切り出した画像23点(すべて2x解像度)
```

CSSの読み込み順は `css/layout.css` → `css/common.css` → `style.css` です。
layout.css / common.css は汎用ファイルのため手を入れず、サイト固有の調整
(要素セレクタ指定の打ち消し等も含む)はすべて `style.css` 側に書いています。

## 実装方針

- カンプが1600px幅の1枚デザインのため、`.page`(1600×4305px)をカンプ座標系とした **ピクセルパーフェクト実装** です。
- レイアウトは **flexbox / grid ベース** で構成しています。
  - 横並び・縦並びの実体があるもの(社員写真の列・募集要項/FAQ・フッターナビ・グローバルナビのリスト)は **flex**。
  - 写真やテキストが重なり合うコラージュ(ヒーロー・About・Work・座談会・フッターブランド部)は **grid スタック**(全子要素を `grid-area: 1/1` に重ね、px マージンで位置決め。`minmax(0,1fr)` トラック + 子の `min-width/height: 0` でトラックの膨張を防止)。
  - `position: absolute` は次の用途のみに限定: セクション大枠のカンプ座標へのアンカー / 方眼紙・グネグネ模様などの非インタラクティブな背景装飾 / ボタン内ラベル等の微細要素 / モーダル・SPヘッダー等の固定UI。
  - grid スタックの子に付けている `position: relative`(オフセットなし)は、768〜1599pxのtransform縮小時のラスタライズ結果を従来の絶対配置版と一致させるためのものです。
- 旧・絶対配置版からの書き換えにあたり、375/1600/1920px幅・ドロワー開・モーダル開の各状態でスクリーンショットのピクセル単位比較を行い、**完全一致**を確認しています(768〜1599pxの縮小表示はレイアウト座標は完全一致・アンチエイリアスのサブピクセル差のみ)。
- class名はすべて **キャメルケース** で統一しています。
- 画像はカンプSVGからレイヤー単位で個別に切り出し、**2x解像度**で書き出しています(Retina対応。CSS側で1xサイズ指定)。
- テキストは可能な限りライブテキスト化しています(ナビ・見出し・本文・ボタン・フッター等)。
- 方眼紙背景・白カード・ピル型ボタン・チェック柄プレースホルダーはCSSで再現しています。
- 副次的な改善: セクションが実座標を持つようになったため、ページ内アンカー(#about 等)がPC表示でも正しい位置へスクロールします(旧実装ではセクションが高さ0のため先頭に飛んでいました)。

## ★ 人間の手による対応が必要な箇所(必読)

1. **フォントが確定していません(最重要)**
   カンプのテキストは全てアウトライン化されており元書体が特定できないため、
   近似のGoogle Fontsで代替しています。正式フォントが分かり次第、
   `style.css` の `--fontGothic` / `--fontMaru` を差し替えてください。
   - 本文・ナビ等 → Zen Kaku Gothic New(代替)
   - 英字セクション見出し(About us 等) → Zen Maru Gothic(代替)
   フォント差し替え後は letter-spacing 各所の再調整を推奨します。

2. **Aboutセクションの写真がプレースホルダーです**
   カンプ自体が「img」表記のグレー市松のため、`.aboutPhoto` をCSSで仮組みしています。
   実写真支給後は `.aboutPhoto` を `<img>` に差し替え、`.aboutPhotoLabel`(「img」の文字)を削除してください。

3. **リンク先がすべて仮です**
   下層ページ未支給のため、ボタン・フッターリンク等は `href="#"`(ページ内アンカーのみ実アンカー)です。URL確定後に差し替えてください。

4. **レスポンシブはコーダー側の判断で実装しています(SPカンプ未支給のため)**
   ブレークポイント構成:
   - **1600px〜** … カンプ等倍表示
   - **768〜1599px** … PCデザインをJSで等倍縮小してフィット(ピクセルパーフェクト維持)
   - **〜767px** … SP専用レイアウト(コーダー判断のデザイン)
   SPでの主な変更点: 固定ヘッダー+ハンバーガードロワー / ヒーローはナビ余白を除いた専用画像(`heroVisualSp.jpg`) /
   アイソメ図は横スクロール / 社員写真は横スクロールカルーセル / 座談会カードはCSSで再構成 /
   トンボ・CMYKストリップ等の「版」装飾はSPでは非表示 / 本文の`<br>`はSPで無効化し自然折返し。
   **SPカンプ支給後は `style.css` 末尾の `@media (max-width: 767px)` ブロックを差し替えてください。**

5. **画像内に含まれるテキスト**
   以下はデザイン上の理由(手書き文字・円弧に沿った文字・写真合成)で画像化しています。文言修正時は画像の再書き出しが必要です。
   - FV全体(キャッチコピー「想いを、表現のすべてで」/「SUDA SEIHAN Co., Ltd.」を含む) … `heroVisual.png`
   - ロゴ … `logoSuda.png`
   - アイソメイラスト内のラベル(01 営業〜04 加工・物流 / 総務 / できたもの) … `isomeIllust.png`
   - 社員写真の部署名(円弧文字)と＋アイコン … `member*.png`
   - 「若手座談会 CROSS TALK」タイトル … `crossTalkTitle.png`
   - 付箋2点(「入社後について」「須田製版で働いて良かったことは？」) … `sticky*.png`
   - 下部のトンボ+CMYK表記ストリップ(印刷版風の装飾) … `bottomStrip.png` / `trimTop.png`

6. **カンプ由来の装飾について**
   四隅のトンボ・CMYKカラーバー・「1923-11-01_© SUDA SEIHAN Co., Ltd.2026」表記は
   「印刷の版」を模したデザイン要素と解釈しそのまま実装しています。
   不要な場合は `.trimTop` / `.bottomStrip` を削除してください。
   (© 表記の年号等は運用に合わせてご確認ください)

7. **ホバー演出は仮実装です**
   ボタン類の hover(透過・反転)、社員写真の拡大は指定がないため仮で入れています。
   トンマナに合わせて調整してください。

## モーダル機能(Work process / Interview)

- **Work process**: イラスト上のラベル(01 営業 / 02 制作 / 03 印刷 / 04 加工・物流 / 総務 / できたもの)がリンクになっており、クリックで該当エリアの拡大画像を全画面モーダル表示します。
  - PC / 縮小表示: イラスト上のラベルに透明ホットスポットを重ねています(ホバーでハイライト)
  - SP: ラベルは小さくタップしづらいため、**イラスト下にチップ型リンクを設置**しています(同じ要素をCSSで切替)
- **Interview**: 社員の円形写真をクリックすると、支給SVGから抽出した**元写真(4528×3016 → 1600px幅に最適化)**を全画面モーダル表示します。
- モーダルは overlay クリック / ×ボタン / Esc キーで閉じられます。`body` スクロールはモーダル表示中ロックされます。
- モーダルのDOMは `.pageScaler`(transform縮小の親)の**外**に置いています。transform配下に置くと `position: fixed` が効かなくなるため、移動しないでください。
- モーダル画像とキャプションの対応は `js/script.js` の `MODAL_DATA` で管理しています。リンク先ページができた際はモーダルではなく遷移に差し替えることも可能です。

## グネグネ模様(テクスチャ版・画面端まで延伸)について

- 追加支給SVG(アートボード2)から、シアン/マゼンタ/イエローのグネグネ模様6点を**粒状テクスチャ付きの個別レイヤー**として切り出しました(輝度マスクのテクスチャはChromiumレンダリングで忠実に再現)。
  - グリッド帯:`squiggleBlue / squiggleYellow / squigglePink`
  - ヒーロー:`squiggleHeroPink / squiggleHeroYellow / squiggleHeroCyan`
- **1600pxを超える画面でも模様が常に画面端まで届く**よう、端が平らにカットされている辺にテクスチャを引き継いだ延長帯(+1200pxずつ)を合成した `〜X.png` を配置しています(横3840pxのモニタまで対応)。横方向のはみ出しは `.pageScaler` がビューポート幅でクリップするため、横スクロールは発生しません。
- これに伴い**ヒーローを分解**しました:`heroSky.png`(半透明の空+波) → グネグネ3レイヤー → `heroCut.png`(人物・手書きコピー・SUDA表記等 ※旧フラット画像から彩度3色をカラーキー抽出)の重ね順です。**SPは軽量なフラット画像(`heroVisualSp.jpg`、テクスチャ版で再生成)のまま**です。
- SP用の装飾背景(`about/work/memberList`)は延長なし版(`squiggle*.png`)を参照しています。
